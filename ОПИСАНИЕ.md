# DXF-kimi25 — полное описание функционала

## 1. Назначение программы

DXF-kimi25 — это система для подготовки задач лазерной резки на базе DXF:

- просмотр и анализ DXF-файлов;
- расчёт технологических метрик (врезки, длина реза и т.д.);
- автоматическая раскладка деталей по листам (nesting);
- экспорт результатов в DXF/CSV;
- обмен отдельными листами раскладки через короткие hash-коды;
- Telegram-бот для получения раскладок по hash-кодам;
- облачная библиотека файлов с каталогами (для авторизованных пользователей).

Проект состоит из нескольких пакетов: `ui-app`, `api-service`, `core-engine`, `bot-service`, `pricing`.

## 2. UI-приложение (web-интерфейс)

### 2.1 Загрузка и просмотр DXF

- Открытие `.dxf` через кнопку, drag-and-drop и быстрые действия интерфейса.
- Работа с несколькими файлами одновременно.
- Переключение активного файла в списке.
- Включение/исключение файла из расчётов через чекбокс.
- Удаление файла из набора.
- Прогресс загрузки/обработки.

### 2.2 Навигация по чертежу

- Масштабирование колесом мыши.
- Панорамирование (перемещение) холста.
- Быстрое «вписать в экран».
- Переключаемая сетка.
- Строка статуса с координатами курсора, зумом, числом сущностей, метриками.

### 2.3 Инспектор объектов

- Выбор сущности кликом на холсте.
- Показ параметров выбранного объекта (тип, геометрия, слой и др.).
- Подсветка при наведении и выборе.

### 2.4 Отображение врезок

- Переключатель **«Врезки»** в нижней части сайдбара.
- Маркеры точек врезки отображаются только для **активного** файла (не всех загруженных).
- Маркеры обновляются при смене активного файла.

### 2.5 Отображение размеров

- Переключатель **«Размеры»** в нижней части сайдбара (рядом с «Врезки»).
- При включении рисуются габаритные размеры документа:
  - горизонтальная размерная линия со стрелками и подписью ширины (под деталью);
  - вертикальная размерная линия с подписью высоты (слева, повёрнута 90°);
  - выносные линии пунктирные, подписи на тёмном фоне.
- Текст и стрелки рисуются в экранных координатах — не зеркалятся при Y-инверсии.
- При наведении мыши на любой объект в режиме размеров — всплывающий tooltip с `W` и `H` конкретной сущности, следует за курсором.
- Не отображается если деталь меньше 60 px на экране.
- Метки адаптируются к масштабу: мкм / мм / м.

### 2.6 Расчёт резки

Для выбранных файлов рассчитываются:

- количество врезок (pierces);
- суммарная длина реза;
- статистика по слоям;
- цепочки реза, открытые/замкнутые контуры.

Есть локальный и API-режим вычислений с индикатором режима (fallback на локальный при недоступности API).

### 2.7 Nesting (раскладка)

- Выбор листа из пресетов или пользовательского размера.
- Настройка зазора.
- Количество копий для каждой детали (в т.ч. быстрые кнопки +/-).
- Автопересчёт при изменениях параметров/количеств.
- Режимы:
  - **Точно** (MaxRects + multi-start, без common-line);
  - **Общий рез** (MaxRects + multi-start, common-line включён).
- Повороты ограничены углами **0°/90°**.

### 2.8 Метрики результата раскладки

Карточки результата показывают:

- число листов;
- средний процент заполнения;
- итоговые врезки;
- итоговую длину реза.

При common-line метрики пересчитываются с учётом экономии:

- вычитание `sharedCutLength` из общей длины реза;
- вычитание `pierceDelta` из общего числа врезок;
- отдельные карточки экономии при наличии реальной экономии.

### 2.9 Визуализация раскладки

- Отрисовка всех листов на одном canvas.
- Реальные контуры деталей на листах.
- Лупа/zoom popup для детального просмотра листа.
- Кнопка закрытия режима раскладки.

### 2.10 Экспорт из UI

- Экспорт всей раскладки в DXF.
- Экспорт данных раскладки в CSV.
- Экспорт **каждого листа отдельно** (кнопка на каждом листе).
- Экспорт **всех листов отдельными DXF** одной командой.

### 2.11 Hash-коды листов

После расчёта раскладки UI отправляет результат на `POST /api/nesting/share` и получает hash-коды листов.

В интерфейсе доступны:

- hash рядом с каждым листом;
- кнопка копирования hash у каждого листа;
- кнопка «Копировать все коды» в панели результатов;
- компактная верхняя кнопка-иконка «Копировать все коды» рядом с кнопкой закрытия раскладки.

### 2.12 Аутентификация через Telegram

- Кнопка **«Войти»** в тулбаре запускает Telegram OAuth-flow.
- После входа UI получает `sessionToken` и `workspaceId`, хранит в `localStorage`.
- Сессия восстанавливается при перезагрузке страницы (`/api/auth/me`).
- Кнопка **«Выйти»** инвалидирует сессию на сервере и очищает localStorage.
- Гостевой черновик (загруженные до входа файлы) сохраняется в localStorage и мигрирует в облако при входе.

### 2.13 Библиотека файлов (workspace)

Доступна только авторизованным пользователям:

- Каталоги: создание, переименование, удаление.
- Файлы: загрузка в каталог, переименование, удаление, перемещение между каталогами.
- Фильтрация списка файлов по выбранному каталогу.
- Кнопка «Выбрать все» применяется к файлам текущего фильтра.
- Лимит: до 200 файлов и 20 каталогов на workspace.
- Состояние чекбоксов (включён/выключен) и количество копий синхронизируются с сервером.

## 3. API-сервис

### 3.1 Основные endpoint-ы

- `GET /health`
- `POST /api/parse`
- `POST /api/normalize`
- `POST /api/cutting-stats`
- `POST /api/nest`
- `POST /api/export/dxf`
- `POST /api/export/csv`
- `POST /api/price`
- `POST /api/bot/message` (служебный, защищён `INTERNAL_API_SECRET`)

### 3.2 Аутентификация

- `GET /api/auth/telegram/login-url` — генерирует одноразовый код и возвращает ссылку на бота.
- `POST /api/auth/telegram/exchange` — обменивает код на sessionToken (rate limit: 10 попыток / 5 мин на IP).
- `GET /api/auth/me` — проверка текущей сессии.
- `POST /api/auth/logout` — инвалидация сессии.

### 3.3 Библиотека (workspace library)

- `GET /api/library` — дерево каталогов и файлов workspace.
- `POST /api/library/catalogs` — создание каталога.
- `PATCH /api/library/catalogs/:id` — переименование.
- `DELETE /api/library/catalogs/:id` — удаление.
- `POST /api/library/files` — загрузка файла.
- `PATCH /api/library/files/:id` — обновление метаданных (имя, checked, quantity, catalogId).
- `DELETE /api/library/files/:id` — удаление.
- `POST /api/library-files-check-all` — массовое обновление чекбоксов.

### 3.4 Endpoint-ы hash-sharing

- `POST /api/nesting/share` — создаёт hash для каждого листа, лимит 50 листов за запрос.
- `GET /api/nesting/sheet/:hash` — возвращает DXF листа по hash.
- `GET /api/nesting/sheet/:hash/info` — JSON-информация о листе.

### 3.5 Хранилище hash-кодов

- In-memory `Map` (`sharedSheetStore`), cap 2 000 записей (вытесняет старые при переполнении).
- TTL записей: 7 дней.
- Опционально: Supabase для персистентного хранения.

### 3.6 Безопасность и rate limiting

- Rate limit на тяжёлые операции (`/api/nest`, `/api/parse` и др.): 10 req / 15 мин на IP.
- Rate limit на code exchange: 10 попыток / 5 мин на IP.
- CORS: разрешены явные origins или `*.vercel.app` в non-production.
- Все входные данные валидируются (UUID, размеры имён, размер DXF payload до 200 МБ).
- Webhook Telegram защищён `X-Telegram-Bot-Api-Secret-Token`.

## 4. Core-engine

Ядро выполняет:

- парсинг DXF (группы, сущности, блоки, слои, атрибуты);
- нормализацию сущностей в `flatEntities`;
- вычисление cutting stats (цепочки, врезки, длина реза);
- раскладку `nestItems`;
- экспорт раскладки в DXF/CSV;
- 2D-рендеринг на Canvas с камерой (zoom/pan/rotate), R-tree для hit-testing.

Ключевые особенности раскладки:

- стратегии BBox (`blf_bbox`, `maxrects_bbox`);
- multi-start выбор лучшей попытки;
- common-line оценка общей кромки и потенциальной экономии;
- при common-line эффективный `gap = 0`.

## 5. Telegram-бот

### 5.1 Основной функционал

- Приём DXF-документа из Telegram (лимит 20 МБ).
- Анализ файла (врезки/длина реза).
- Генерация и отправка JPEG-превью.
- Меню настроек (лист, зазор, количество, режим).
- Запуск раскладки прямо в боте (с защитой от таймаута 25 с).
- Экспорт варианта раскладки в DXF/CSV из бота.

### 5.2 Hash-сценарий

Если в текстовом сообщении есть hash (или несколько hash, формат 8 hex-символов), бот:

- ищет их в `sharedSheetStore`;
- формирует DXF каждого найденного листа;
- отправляет документы пользователю;
- сообщает о hash, которые не найдены/истекли.

### 5.3 Telegram OAuth (авторизация UI)

- Бот принимает команду с одноразовым кодом от пользователя.
- Подтверждает код в API (`POST /api/auth/telegram/confirm`).
- UI завершает обмен кодом через `POST /api/auth/telegram/exchange`.

## 6. Pricing

Модуль расчёта стоимости (`packages/pricing`), доступен через `POST /api/price`.  
Параметры: длина реза, число врезок, число листов, коэффициент сложности.  
Возвращает итоговую стоимость и разбивку по статьям (в RUB).

## 7. Развёртывание и ограничения

### 7.1 Локальный / серверный запуск

Рекомендуется для полного функционала: hash-sharing, Telegram-бот и API работают в одном процессе с общей памятью.

### 7.2 Vercel (serverless)

- API работает как serverless-функции.
- Telegram polling не подходит — нужен webhook.
- In-memory store не гарантирует постоянство между инстансами; для надёжного hash-sharing используется Supabase.

### 7.3 Переменные окружения

| Переменная | Назначение |
|---|---|
| `INTERNAL_API_SECRET` | Защита служебных endpoint-ов |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота |
| `TELEGRAM_WEBHOOK_URL` | URL webhook для регистрации |
| `TELEGRAM_WEBHOOK_SECRET` | Секрет валидации входящих от Telegram |
| `SUPABASE_URL` | Supabase проект URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Ключ Supabase (service role) |

## 8. Пользовательский сценарий (end-to-end)

1. Пользователь открывает UI, при необходимости входит через Telegram.
2. Загружает DXF-файлы (локально или из облачной библиотеки).
3. Проверяет метрики резки, опционально включает отображение врезок и размеров.
4. Запускает раскладку, получает листы с метриками.
5. Скачивает нужные листы DXF или все листы одной командой.
6. Копирует hash-коды листов и передаёт оператору.
7. Оператор отправляет hash в Telegram-бот.
8. Бот возвращает соответствующие DXF-листы.

## 9. История изменений (основные вехи)

- **Базовый функционал:** парсинг DXF, просмотр, cutting stats, nesting, экспорт.
- **Hash-sharing:** per-sheet DXF экспорт, hash-генерация, API share/get-by-hash, бот по hash.
- **Common-line метрики:** корректный пересчёт экономии реза и врезок.
- **Telegram OAuth:** вход через бота, сессии, восстановление при перезагрузке.
- **Workspace library:** каталоги, облачные файлы, синхронизация состояния.
- **Security hardening:** rate limiting, input validation, XSS-защита, cap in-memory maps, CORS политика.
- **Врезки fix:** точки врезки показываются только для активного файла.
- **Отображение размеров:** переключатель «Размеры», габаритный bbox с подписями W×H, hover tooltip.
