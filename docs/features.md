## Веб-приложение

### Просмотр DXF-файлов
- Загрузка DXF-файлов через кнопку или drag & drop
- Рендеринг всех основных типов сущностей: LINE, ARC, CIRCLE, SPLINE, ELLIPSE, POLYLINE, LWPOLYLINE, INSERT (блоки)
- Масштабирование, панорамирование, подгонка под экран
- Отображение статистики файла: количество проколов, длина реза, замкнутые контуры, открытые пути — по слоям
- Переключение видимости слоёв
- Миниатюры (thumbnails) файлов в библиотеке

### Интерфейс
- Двуязычный интерфейс: **русский / английский** (переключение в один клик)
- Тёмная тема
- Адаптивная вёрстка

## Telegram-бот

### Авторизация через Telegram
- Вход в веб-приложение через Telegram Mini App (TMA) — автоматическая авторизация при открытии из Telegram
- Вход по одноразовому коду: бот присылает 6-символьный код, пользователь вводит его в браузере
- Сессия привязывается к Telegram user ID и рабочему пространству

### Команды бота
- `/start` — приветствие, отображение главного меню
- `/login` — генерация одноразового кода для входа в веб-приложение
- Получение и обработка сообщений от пользователя через webhook

### Webhook
- Автоматическая регистрация webhook при старте сервера (если заданы `TELEGRAM_BOT_TOKEN` и `TELEGRAM_WEBHOOK_URL`)
- Ручная регистрация webhook через API: `POST /api/telegram/webhook/register`
- Проверка секрета webhook (`x-telegram-bot-api-secret-token`)

## API

### DXF-обработка

| Endpoint | Описание |
|---|---|
| `POST /api/parse` | Парсинг DXF-файла, возврат нормализованного документа |
| `POST /api/normalize` | Нормализация DXF: сглаживание сплайнов, объединение контуров |
| `POST /api/cutting-stats` | Статистика резки: длина реза, проколы, контуры — опционально по слоям |
| `POST /api/nest` | Раскладка деталей на листе (nesting) |
| `POST /api/price` | Расчёт стоимости резки |
| `POST /api/nesting-share` | Сохранение результата раскладки по хешу |
| `GET /api/nesting-share/:hash` | Получение сохранённой раскладки по хешу |

### Форматы загрузки DXF
- Base64 в JSON-теле
- Multipart/form-data
- Бинарный поток (`application/octet-stream`) до 200 МБ

### Health check
- `GET /health` или `GET /api/health` — статус сервера

## Аутентификация

### Методы входа
- **Telegram Mini App (TMA)** — автоматический вход при открытии в Telegram, валидация `initData` с подписью HMAC-SHA256
- **Код авторизации** — бот генерирует одноразовый 6-символьный код, действителен 5 минут
- **Bearer-токен** — принятие существующей сессии через заголовок `Authorization`

### Сессии
- Сессии хранятся в Supabase (таблица `auth_sessions`)
- Срок действия сессии — 30 дней
- Выход из системы: `POST /api/auth/logout` — инвалидация сессии и очистка cookie

### Эндпоинты
| Endpoint | Описание |
|---|---|
| `POST /api/auth/tma-init` | Вход через Telegram Mini App |
| `POST /api/auth/telegram/exchange-code` | Обмен кода на сессию |
| `POST /api/auth/adopt-token` | Принятие Bearer-токена, установка cookie |
| `GET /api/auth/me` | Информация о текущей сессии |
| `POST /api/auth/logout` | Выход, удаление сессии |

## Рабочее пространство

> Требует аутентификации. Файлы хранятся в Supabase Storage.

### Каталоги
- Создание, переименование, удаление каталогов
- Перемещение файлов между каталогами (drag & drop)
- Сворачивание/разворачивание каталогов в UI
- Скачивание всех файлов каталога в ZIP-архиве

### Файлы библиотеки
- Загрузка DXF-файлов в библиотеку (до 200 МБ на файл):
  - Multipart upload
  - Base64 upload
  - Прямая загрузка через подписанный URL (signed upload)
- Скачивание исходного файла
- Переименование, удаление
- Отметка файлов как «проверенных» (`checked`)
- Отображение метрик: площадь, длина реза, количество проколов

### Материалы
- Назначение материала каждому файлу: группа → марка → толщина
- База материалов: сталь, нержавейка, алюминий, медь, латунь, титан и др.
- Расчёт веса заготовки по площади и толщине
- Применение материала сразу ко всем файлам в каталоге
- Синхронизация назначений материалов с сервером

### Эндпоинты библиотеки
| Endpoint | Описание |
|---|---|
| `GET /api/library/tree` | Дерево каталогов и файлов |
| `POST /api/library/catalogs` | Создать каталог |
| `PATCH /api/library/catalogs/:id` | Переименовать каталог |
| `DELETE /api/library/catalogs/:id` | Удалить каталог |
| `POST /api/library/files` | Загрузить файл (multipart или base64) |
| `POST /api/library/files/signed-upload` | Получить подписанный URL для загрузки |
| `POST /api/library/files/direct-upload` | Прямая загрузка бинарного файла |
| `PATCH /api/library/files/:id` | Обновить метаданные файла |
| `DELETE /api/library/files/:id` | Удалить файл |
| `GET /api/library/files/:id/download` | Скачать файл |
| `POST /api/library/files/checked` | Установить флаг checked |
| `GET /api/library/files/:id/materials` | Получить материалы файла |
| `PUT /api/library/files/:id/materials` | Назначить материал файлу |

## Раскладка деталей (Set Builder)

### Основные возможности
- Формирование набора деталей (сет) из библиотеки с указанием количества
- Выбор листа из предустановленных форматов или ввод произвольного размера (мм)
- Сохранение пользовательских форматов листов
- Настройка зазора между деталями (мм)

### Режимы раскладки
| Режим | Описание |
|---|---|
| **Обычный** | Стандартная раскладка с зазором |
| **Совмещённый рез (Common Line)** | Соседние детали разрезаются одним проходом лазера, зазор = 0 |

### Параметры нестинга
- **Поворот деталей**: вкл/выкл, шаг поворота 1°, 2° или 5°
- **Multi-start**: многократный запуск с разными начальными условиями для лучшего результата
- **Seed**: фиксированное зерно генератора для воспроизводимых результатов
- **Common-line**: максимальное расстояние слияния (мм), минимальная длина общего отрезка (мм)

### Результаты раскладки
- Отображение листов с размещёнными деталями
- Процент заполнения каждого листа
- Метрики: общая длина реза, длина совмещённого реза, экономия реза, количество проколов
- Миниатюры листов с раскладкой
- Предпросмотр отдельного листа с деталями

### Ручная корректировка
- Режим ручного перетаскивания деталей на листе (drag mode)
- Авто-перераскладка отдельного листа (`Auto Arrange`) с новым случайным seed
- Пересохранение листа с учётом ручных перемещений

### Экспорт
- Экспорт отдельного листа в DXF
- Экспорт всех листов в DXF (по одному файлу)
- Копирование хеша листа для передачи в бот
- Копирование всех хешей сразу

### Сортировка и фильтрация библиотеки
- Поиск по названию
- Фильтр по каталогу
- Сортировка по: названию, площади, количеству проколов, длине реза (возр./убыв.)
- Массовые операции: добавить в сет, удалить из библиотеки, задать количество

### Персистентность
- Состояние сет-билдера сохраняется в localStorage
- Материалы синхронизируются с сервером для авторизованных пользователей
- Локальный кэш библиотеки для быстрого первого отображения

## Оптимизатор DXF

### Одиночный файл
Анализ и оптимизация DXF-файла по правилам:

| Код | Правило |
|---|---|
| R1 | Удаление дублирующихся сущностей |
| R4 | Удаление нулевых отрезков |
| R5 | Удаление микро-сегментов (< 0.1 мм) |
| R6 | Предупреждение о сплайнах и эллипсах (плохо для резки) |

- Предпросмотр файла с цветовой подсветкой проблемных элементов (красный — критично, жёлтый — предупреждение)
- Применение выбранных правил оптимизации
- Скачивание оптимизированного DXF
- Скачивание отчёта в JSON

### Пакетная оптимизация (Batch Optimizer)
- Оптимизация всех файлов каталога или всей библиотеки за один раз
- Выбор правил оптимизации и точности (epsilon, мм)
- Прогресс обработки с возможностью отмены
- Скачивание всех оптимизированных файлов в ZIP-архиве

### Разделение DXF (Split)
- Разбивка DXF-файла на отдельные замкнутые контуры
- Каждый контур сохраняется как отдельный файл в библиотеке

## Ограничения и лимиты

### Rate limiting
| Эндпоинт | Лимит |
|---|---|
| `POST /api/nest` | 10 запросов / минута / IP |
| `POST /api/price`, `/api/parse`, `/api/normalize`, `/api/cutting-stats` | 10 запросов / минута / IP |
| `POST /api/auth/tma-init` | 20 запросов / минута / IP |
| `POST /api/auth/telegram/exchange-code` | 10 запросов / 5 минут / IP |

### Размеры файлов
| Ограничение | Значение |
|---|---|
| DXF через API (`/api/parse`, `/api/nest` и др.) | до 50 МБ |
| Загрузка в библиотеку (multipart / direct) | до 200 МБ |
| Сохранение раскладки (`/api/nesting-share`) | до 20 МБ |
| Остальные запросы | до 1 МБ |

### Нестинг
- Максимум 500 деталей в одном запросе
- Максимум 10 000 штук одной детали
- Таймаут нестинга: 30 секунд
- Размер листа: от 1 до 100 000 мм по каждой стороне

### Библиотека
- Максимальное количество файлов в каталоге: 10 000
- Ручная корректировка раскладки: до 120 деталей на листе
