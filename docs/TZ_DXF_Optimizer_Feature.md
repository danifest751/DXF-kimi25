# ТЗ: DXF Optimizer (анализ + оптимизация “грязных” DXF) для раскладки/лазерной резки

Версия: 1.0  
Статус: Draft  
Цель: реализовать модуль, который **читает DXF**, показывает **что внутри**, выявляет проблемы “под резку” и позволяет **безопасно оптимизировать** файл (уменьшить число сущностей/вершин, убрать мусор/дубли, улучшить контуры), не ломая геометрию.

---

## 1) Цели и принципы

### 1.1 Основные цели
1) Уменьшить “грязь” DXF: дубли, микросегменты, нулевые сущности, лишние слои/аннотации.  
2) Улучшить пригодность под резку: корректные замкнутые контуры, меньше ошибок в CAM/нестинге.  
3) Снизить вычислительную нагрузку на анализ/нэйстинг/прайсинг.  
4) Дать пользователю **прозрачность**: отчёт “что изменилось” и “почему”.

### 1.2 Главный принцип безопасности
- **Исходник никогда не изменяется**.
- Оптимизация создаёт **новый DXF** + отчёт.
- Любая операция, меняющая геометрию, выполняется **только при заданном допуске** (tolerance) и помечается как “lossy”.

---

## 2) Термины

- **Entity** — DXF-сущность (LINE, ARC, LWPOLYLINE, SPLINE, TEXT, INSERT, …).
- **Contour** — связанный набор примитивов, образующий путь реза.
- **Closed contour** — замкнутый контур (деталь/вырез).
- **Tolerance (ε)** — допуск сравнения/аппроксимации, мм.
- **Optimizer Plan** — выбранный набор правил оптимизации с параметрами.

---

## 3) Сценарии использования (User Stories)

### US-01: Диагностика DXF
Пользователь загружает DXF → видит:
- типы сущностей и количество
- слои и распределение по слоям
- предупреждения/ошибки пригодности под резку
- оценку “DXF Health Score”

### US-02: Рекомендации по оптимизации
Пользователь открывает вкладку “Оптимизация” → видит список рекомендованных правил (чекбоксы), параметры допусков и прогноз “что изменится”.

### US-03: Предпросмотр “до/после”
Пользователь нажимает Preview → видит overlay сравнение (diff): удалённое/объединённое/упрощённое подсвечено.

### US-04: Экспорт оптимизированного DXF
Пользователь нажимает Export → получает новый DXF + отчёт (JSON/PDF/HTML).

### US-05: Использование оптимизированного DXF в пайплайне
Пользователь может отправить оптимизированный DXF сразу в:
- анализ резки
- прайсинг
- нэйстинг

Разделы:

1) **Overview**
- Health Score (0–100)

- Кнопки:
  - Export Optimized DXF
  - Export Report (JSON/HTML/PDF)

## 5) Правила оптимизации (Rules)

### 5.1 Safe (без изменения геометрии)
**R1: Remove Zero-Length Entities**
- Удалять линии длиной 0
- Удалять дуги/окружности с радиусом 0
- Удалять полилинии с 1 точкой

**R2: Remove Non-Cutting Entities (optional)**
- Удалять/игнорировать: TEXT/MTEXT, DIMENSION, LEADER, HATCH (настраиваемо)
- По умолчанию: только помечать, не удалять (пользователь выбирает)

**R3: Normalize Units**
- Определить/задать единицы (INSUNITS)
- Привести в мм (если в настройках проекта мм)
- Записать факт конвертации в отчёт

**R4: De-duplicate Identical Entities (ε compare)**
- Удалять дубли линий/дуг/полилиний с допуском ε (по умолчанию 0.01 мм)
- Дубли в разных слоях: настраиваемо (по умолчанию чистим только в пределах одного “cut layer group”)

**R5: Merge Collinear Segments**
- Если 2+ LINE коллинеарны и касаются (gap ≤ ε), объединить в одну LINE
- Параметры:
  - ε_gap: 0.01 мм
  - angle_tol: 0.1°

**R6: Join Connected Primitives into Polyline (lossless where possible)**
- Склеить цепочки LINE/ARC в LWPOLYLINE с bulge, если:
  - последовательность непрерывна (gap ≤ ε)
  - дуги корректно конвертируются в bulge без аппроксимации
- Это не меняет геометрию, но уменьшает количество сущностей

### 5.2 Semi-safe (в пределах допуска; lossy)
**R7: Simplify Polylines (RDP)**
- Упростить LWPOLYLINE, уменьшая число вершин
- Параметр: max_dev_mm (по умолчанию выключено; рекомендовано 0.05 мм)
- Требование: отчёт о максимальном отклонении

**R8: Convert SPLINE/ELLIPSE to ARCs/Polylines**
- SPLINE → набор ARC или полилиния
- ELLIPSE → набор ARC или полилиния
- Параметры:
  - max_dev_mm (например 0.05–0.1 мм)
  - max_seg_len_mm (ограничение длины сегмента)

### 5.3 Risky (repair; только с подтверждением)
**R9: Close Small Gaps**
- Если контур почти замкнут (gap ≤ threshold), замкнуть
- Параметр: close_gap_mm (например 0.05–0.2 мм)
- Требование: показывать diff и список закрытых мест

**R10: Remove Construction Geometry by Rules**
- Удалять “строительные” элементы по:
  - слоям (например CONSTR, DIM)
  - цвету/linetype
- По умолчанию выключено

## 6) Детектор проблем (Diagnostics)

Оптимизатор обязан находить и показывать:

### Critical
- нечитабельный/повреждённый DXF
- экстремально большие координаты / неверный масштаб (детали “в километрах”)
- self-intersections на контуре (если детектируется)
- open contours там, где ожидается closed (если есть признак детали)

### Warning
- SPLINE/ELLIPSE присутствуют (CAM может не принять)
- много коротких сегментов (micro-segmentation)
- дубли (same line repeated)
- контуры с gap (почти замкнутые)
- слои с примесями (cut + annotation)

### Info
- текст/размеры/штриховки
- блоки/insert’ы

---

## 7) Алгоритмические требования

### 7.1 Нормализация
Перед любыми сравнениями:
- приведение координат к float64
- нормализация направления (например сортировка концов линий)
- canonical form для сравнения

### 7.2 Spatial index
Для дедупа/поиска соседей:
- grid hash / R-tree
- цель: дедуп и merge работают не O(n²), а близко к O(n log n)

### 7.3 Контурный граф
Для join/close gaps:
- строить граф точек (кластеризация точек по ε)
- извлекать цепочки и петли

---

## 8) Выходные артефакты

### 8.1 Optimized DXF
- новый файл, имя: `{original}_optimized.dxf`
- сохранять структуру слоёв по умолчанию (или опция “flatten layers”)

### 8.2 Report (обязателен)
Формат: JSON (обязателен), HTML/PDF (опционально)

JSON должен содержать:
- входные параметры (ε, включенные правила)
- before/after счётчики:
  - total_entities, total_vertices
  - by_type counts
  - by_layer counts
  - closed_contours, open_paths
- список применённых операций:
  - rule_id, affected_count
  - max_deviation_mm (если lossy)
- список проблем (issues) и их статус (resolved/remaining)

---

## 9) Настройки по умолчанию (MVP)

- compare epsilon ε = **0.01 мм**
- merge angle tolerance = **0.1°**
- join gap tolerance = **0.01 мм**
- RDP simplify выключено
- SPLINE conversion выключено
- close gaps выключено
- remove non-cut entities: “prompt user”

MVP пресет “Laser Safe” включает:
- R1, R4, R5, R6 (+ опционально R2)

---

## 10) Нефункциональные требования

### 10.1 Производительность
- DXF до 50MB: inventory ≤ 5–10 сек (на типовом ПК)
- оптимизация safe-пакета ≤ 30–60 сек
- preview диффа интерактивен (грубая отрисовка допустима)

### 10.2 Надёжность
- идемпотентность для одинакового входа и параметров
- ошибки сохраняются в отчёте с кодом и описанием

### 10.3 UX
- всегда показывать прогресс (особенно на больших файлах)
- предупреждать о lossy операциях

## 11) Acceptance Criteria (приёмка)

### MVP
1) Загрузка DXF → отображение Inventory (типы сущностей, слои).
2) Показываются Issues (critical/warn/info) с подсветкой на canvas.
3) Работают safe правила: R1, R4, R5, R6.
4) Export создаёт optimized DXF + JSON report с before/after.
5) Исходник не изменяется.
6) После оптимизации файл корректно проходит анализ резки и nesting.

### Extended
7) Работают lossy правила R7/R8 с допуском и отчётом отклонения.
8) Preview Before/After/Overlay показывает изменения визуально.

## 12) План реализации (рекомендуемая последовательность)

1) Inventory + Issues (DXF health)
2) Safe pipeline (R1 + R4 + R5)
3) Join primitives (R6) + contour graph
4) Export optimized DXF + JSON report
5) Preview diff overlay
6) Lossy пакет: simplify + spline conversion
7) Repair пакет: close gaps (если нужно)

---

## 13) Риски и ограничения

- DXF из разных CAD имеет разные особенности; нужен парсер с хорошим покрытием.
- Упрощение геометрии меняет деталь — lossy режим только явный.
- SPLINE conversion может раздувать число сегментов — нужны лимиты max_seg_len/max_segments.
