## 1. Анализ текущих проблем

### 1.1 Перегруженный Toolbar

**Сейчас** (строки 939–993 index.html):
```
[Logo] | [Открыть] [Вписать] | [Свойства] [Раскладка] [Экспорт] [Сетка] | [Врезки] | [badge: 12 / 4.82м] | stats
```

**Проблемы**:
- **10 элементов** в одной строке — перегрузка
- Кнопка «Экспорт» скрыта (`display:none`) и появляется магически — пользователь не знает о ней
- «Врезки» (pierce toggle) — это переключатель визуализации, но выглядит как кнопка действия
- Cutting badge (`12 врезок · 4.82м`) дублирует информацию из status bar
- На планшете (≤1024px) текст кнопок скрывается через `font-size: 0` — костыль

### 1.2 Панель раскладки (Nesting Panel) — главная проблема

**Сейчас** (строки 1049–1166 index.html): **8 секций** в одной колонке 300px:
1. Лист металла (select + custom inputs)
2. Зазор (input)
3. Поворот (checkbox + select шага)
4. Алгоритм (select режима + select стратегии + checkbox multistart + input seed + checkbox common-line + status + input Δmerge + input minLen)
5. Детали и количество (динамический список)
6. Кнопка «Разложить»
7. Результат (6 карточек в grid 2×3)
8. Экспорт (DXF + CSV)

**Проблемы**:
- **Секция «Алгоритм»** — 9 контролов в одной секции. Пользователь видит: режим, стратегию, multistart, seed, common-line, status, Δmerge, minLen. Это перегрузка для 90% пользователей
- Режим (`fast/precise/common`) и стратегия (`blf_bbox/maxrects_bbox`) — **дублирование**. Режим уже определяет стратегию, но оба видны
- `Seed` — нужен только для отладки, но занимает место наравне с основными параметрами
- `Δ merge` и `Min len` — экспертные параметры, видны всегда
- Результат (6 карточек) — слишком много для 300px ширины. Карточки мелкие, текст нечитаем
- Нет визуального разделения между «настройками» и «результатом»

### 1.3 Монолитный main.ts

**1421 строк** в одном файле:
- DOM-элементы (строки 85–151) — **67 переменных** `getElementById`
- Состояние (строки 152–167) — глобальные переменные
- Загрузка файлов (строки 189–298)
- Статистика (строки 308–341)
- Рендер списка файлов (строки 345–380)
- Drag & Drop (строки 382–393)
- Навигация мышью (строки 395–462)
- Кнопки (строки 464–484)
- Инспектор (строки 491–527)
- Nesting логика (строки 535–866)
- Nesting рендер (строки 892–1052)
- Zoom popup (строки 1060–1358)
- Адаптивность (строки 1360–1407)
- Клавиатура (строки 1409–1421)

**Проблемы**:
- Невозможно поддерживать и расширять
- Нет разделения на модули
- 67 DOM-переменных в глобальном scope

### 1.4 Дублирование информации

| Данные | Где показаны |
|--------|-------------|
| Врезки | toolbar badge, status bar, nesting results |
| Длина реза | toolbar badge, status bar, nesting results |
| Кол-во файлов | sidebar header, stats span |
| Режим вычислений | floating badge (правый нижний угол) |

### 1.5 Скрытые возможности

Пользователь **не знает** о:
- `Shift+Click` = pan (нигде не написано)
- `F` = zoom to fit
- `G` = toggle grid
- `Ctrl+O` = открыть файл
- `Escape` = закрыть раскладку / снять выделение
- Колёсико на листе раскладки = zoom popup

## 2. Предложение по переработке

### 2.1 Принцип: Progressive Disclosure

Показывать **минимум** по умолчанию, раскрывать детали по запросу:

```
Уровень 1 (всегда видно):     Режим, Лист, Количество, Запустить
Уровень 2 (по клику «Ещё»):   Зазор, Поворот, Common-line вкл/выкл
Уровень 3 (по клику «Эксперт»): Стратегия, Seed, MultiStart, Δmerge, MinLen
```

### 2.2 Новый Toolbar — 2 зоны

```
┌──────────────────────────────────────────────────────────────────┐
│  DXF Viewer  │  [📂 Открыть] [⊞ Вписать]  │  [📐 Раскладка]   │
│              │                              │                    │
│              │  ← Зона файлов               │  ← Зона анализа   │
└──────────────────────────────────────────────────────────────────┘
```

**Убрать из toolbar**:
- Cutting badge → перенести в sidebar (под списком файлов)
- Pierce toggle → перенести в sidebar (рядом со статистикой)
- Кнопку «Сетка» → в контекстное меню canvas
- Кнопку «Свойства» → инспектор открывается по клику на объект
- Кнопку «Экспорт» → в панель результатов раскладки
- Stats span → в sidebar

**Итого в toolbar**: Logo + Открыть + Вписать + Раскладка = **4 элемента** вместо 10.

## 3. Рефакторинг кода

### 3.1 Разбить main.ts на модули

```
src/
├── main.ts              ← точка входа, инициализация
├── state.ts             ← глобальное состояние (loadedFiles, activeFileId, etc.)
├── ui/
│   ├── toolbar.ts       ← toolbar кнопки и логика
│   ├── sidebar.ts       ← sidebar файлов + статистика
│   ├── inspector.ts     ← инспектор свойств
│   ├── nesting-panel.ts ← панель раскладки (настройки + результат)
│   ├── statusbar.ts     ← status bar
│   └── shortcuts.ts     ← keyboard shortcuts + help popup
├── canvas/
│   ├── navigation.ts    ← pan, zoom, mouse events
│   ├── nesting-view.ts  ← рендер листов раскладки
│   └── zoom-popup.ts    ← popup при наведении на лист
├── api.ts               ← apiPostJSON, apiPostBlob
└── file-loader.ts       ← загрузка, drag&drop, парсинг DXF
```

**Выигрыш**: каждый файл 80–150 строк вместо 1421 в одном.

### 3.2 Вынести CSS из index.html

Сейчас **933 строки CSS** inline в `<style>`. Вынести в:
```
src/
├── styles/
│   ├── base.css         ← переменные, reset, scrollbar, body
│   ├── toolbar.css
│   ├── sidebar.css
│   ├── canvas.css
│   ├── nesting.css
│   ├── statusbar.css
│   ├── responsive.css   ← media queries
│   └── animations.css
```

Импортировать через Vite: `import './styles/base.css'` в main.ts.

### 3.3 Убрать дублирование рендера

Функции `renderAllNestingSheets` (строки 892–1052) и `renderZoomSheet` (строки 1071–1199) содержат **80% одинакового кода** (рисование деталей на листе). Вынести общую функцию:

```typescript
function renderSheetToContext(
  ctx: CanvasRenderingContext2D,
  sheet: NestingSheet,
  result: NestingResult,
  options: { scale: number; ox: number; oy: number; colorMap: Map<number, number> }
): void { ... }
```
