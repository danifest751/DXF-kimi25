# Предложения Cascade по улучшению UX левой панели

Изучив текущую реализацию (DOM в `index.html`, логику рендера в `main.ts` и стили в `sidebar.css`) и ознакомившись с предыдущим документом, я сформировал собственное видение того, как можно сделать панель управления по-настоящему удобной, современной и решающей проблемы интерфейса.

---

## 1. Контекстная шапка и «умные» действия

**Проблема:** Кнопки «Выделить все», «+ Каталог» и «+ DXF» занимают много места, конкурируют за внимание и не зависят от контекста (гость/авторизован, пусто/есть файлы).

**Решение: Контекстный Header**
- Заменить текстовые кнопки на иконки с tooltip'ами для экономии места.
- Основное CTA-действие всегда справа (Upload).
- Кнопку «+ Каталог» (иконка папки с плюсом) показывать **только если пользователь залогинен** (проверять `authWorkspaceId`).
- Убрать кнопку «Выделить все» из шапки. Мультивыбор лучше сделать чекбоксом в заголовке списка (как в email-клиентах) или шорткатом (Ctrl+A).

*Визуализация:*
```text
[ Файлы и каталоги ]        [ 📁+ ] [ 📄+ ]
```

---

## 2. Разделение сущностей: Фильтры vs Структура

**Проблема:** Сейчас в `renderFileList()` каталоги и файлы рендерятся в один плоский список `fileListEl`. Чекбоксы каталогов и клики по ним (для фильтрации) неочевидны. Фильтр каталогов (`catalogFilter` с "чипсами") дублирует функционал списка, перегружая интерфейс.

**Решение: Панель навигации (Navigation Tree) + Рабочая область (File View)**

Вместо дублирования логики в чипсах и в рядах списка, перейдем к классическому паттерну проводника:
1. **Зона каталогов (Folders):** Аккордеон или список папок.
   - По умолчанию выбрано «Все файлы».
   - Клик по папке отфильтровывает список файлов ниже.
   - Рядом с активной папкой появляется иконка [⋮] для действий (Переименовать, Удалить).
2. **Зона файлов (Items):** Показывает только файлы, принадлежащие выбранному в зоне каталогов фильтру.

*Визуализация:*
```text
▼ КАТАЛОГИ
  [ Все файлы (15) ]
  [ Без каталога (2) ]
  [ Проект А (8) ]  [⋮]  <-- Активный фильтр, видны действия
  [ Проект Б (5) ]

▼ ФАЙЛЫ
  [☑] Выбрать все в фильтре
  ────────────────────────
  [☑] деталь_1.dxf    [🗑]
  [☐] деталь_2.dxf    [🗑]
```

**Технический профит:** Мы избавляемся от сложной синхронизации `selectedCatalogIds` между `catalogFilter` и `renderFileList`. Будет единый `activeCatalogId` (или `null` для "Все").

## 3. Индикация "Гостевого" режима (Guest Mode)

**Проблема:** Пользователь не понимает, почему он не может создать каталог, если не залогинен. Кнопка просто может выдавать ошибку или отсутствовать без объяснений.

**Решение: Banner "Guest Mode"**
Если `!authSessionToken`, в зоне каталогов вместо списка (или под ним) показывать красивый banner:
```text
[ 💡 Войдите через Telegram, чтобы группировать файлы по каталогам и сохранять их в облаке. ]
[ Кнопка: Войти ]
```
Это работает как онбординг и драйвер регистраций.

---

## 4. Оптимизация статистики (Footer)

**Проблема:** Статистика реза сейчас прибита гвоздями к футеру (`sidebar-footer`), но она относится к *выбранным* файлам.

**Решение: Липкая сводка (Sticky Summary)**
Поместить мини-сводку над списком файлов. Она должна быть видна всегда, когда есть выбранные файлы (`loadedFiles.some(f => f.checked)`).
```text
[ Выбрано: 3 файла | Врезок: 12 | Рез: 5.4м ]
```
Переключатель "Врезки" (Pierce toggle) логичнее перенести в тулбар (к кнопке "Сетка"), так как это настройка *отображения канваса*, а не свойство файлов.

---

## 5. Драг-н-дроп (Drag & Drop) организация

**Проблема:** Перенос файлов в каталоги сейчас (вероятно) не реализован или неудобен.

**Решение: Нативный D&D API**
Сделать элементы `.file-item` перетаскиваемыми (`draggable="true"`), а строки `.catalog-row` — принимающими (`dragover`, `drop`).
- При дропе файла на каталог отправляем API запрос на обновление `catalogId` файла.
- Это стандартный UX, ожидаемый любым инженером.

---

## 6. Быстрые действия (Context Menus & Hover)

**Проблема:** Кнопки "Переим." и "Удалить" занимают место и визуально шумят. Окно `prompt()` для переименования выглядит архаично.

**Решение:**
- **Файлы:** Оставить корзину при hover. Добавить двойной клик по имени для переименования (inline-редактирование, `contenteditable`).
- **Каталоги:** Иконка [⋮] при hover, открывающая маленький popover:
  - ✏️ Переименовать (превращает строку каталога в `input`)
  - 🗑 Удалить
- **Удаление каталога:** Использовать красивый кастомный модальный диалог вместо системного `confirm()`, где явно спросить: "Удалить файлы или перенести в Без каталога?".

---

## Итог: План рефакторинга (Action Item)

Если концепция утверждается, я предлагаю следующий порядок реализации (разбитый на изолированные PR):

1. **Шаг 1: Очистка шапки и Guest Banner.** Перенос кнопок, добавление баннера авторизации для неавторизованных пользователей. Перенос "Врезок" в тулбар.
2. **Шаг 2: Разделение View.** Переписывание `renderFileList`. Удаление `catalogFilter` (чипсов). Создание двух списков: один для навигации по папкам, второй для файлов выбранной папки. Единый чекбокс "Выделить все" над списком файлов.
3. **Шаг 3: Inline редактирование и Popovers.** Замена `prompt` и `confirm` на красивые UI элементы. Добавление меню действий [⋮] для каталогов.
4. **Шаг 4: Drag & Drop.** Реализация перетаскивания файлов между папками.
